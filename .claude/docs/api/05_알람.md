# 🔔 알람 API 명세

---

## 📍 20 알람 조회 API

- **설명:** 인증된 사용자의 알람 목록을 조회하는 API입니다. 피드백 도착, 센서 임계값 초과 등 시스템에서 발생한 알람을 최신순으로 반환합니다. 읽지 않은 알람 수도 함께 반환합니다. (인증 필요)
- **헤더:** `Authorization: Bearer {accessToken}`

**Parameter**

| 필드 이름 | 필수 여부 | 타입 | 설명 |
| --- | --- | --- | --- |
| placeId | 선택 | Int | 특정 장소의 알람만 조회 (미입력 시 전체 조회) |
| page | 선택 | Int | 페이지 번호 (기본값: 0) |
| size | 선택 | Int | 페이지 크기 (기본값: 20) |

**Request Body**

| 필드 이름 | 필수 여부 | 타입 | 설명 |
| --- | --- | --- | --- |

**Request Sample**

```
GET /api/alarm?placeId=501&page=0&size=20
```

**응답코드**

| 성공 여부 | 응답 코드 | 설명 |
| --- | --- | --- |
| 성공 | 200 | 알람 목록 조회 성공 |
| 실패 | 401 | 인증 실패 |
| 실패 | 500 | 서버 내부 오류 |

**Response Body**

| 필드 이름 | 필수 여부 | 타입 | 설명 |
| --- | --- | --- | --- |
| unreadCount | 필수 | Int | 읽지 않은 알람 수 |
| totalCount | 필수 | Int | 전체 알람 수 |
| alarms | 필수 | Array | 알람 목록 |
| alarms[].alarmId | 필수 | Long | 알람 고유 ID |
| alarms[].type | 필수 | String | 알람 유형 (FEEDBACK / SENSOR_ALERT) |
| alarms[].title | 필수 | String | 알람 제목 |
| alarms[].body | 필수 | String | 알람 내용 |
| alarms[].isRead | 필수 | Boolean | 읽음 여부 |
| alarms[].placeId | 필수 | Int | 관련 장소 ID |
| alarms[].createdAt | 필수 | DateTime | 알람 생성 시간 (ISO 8601) |

**JSON 형식 응답**

```json
{
  "success": true,
  "message": "API 호출 성공",
  "data": {
    "unreadCount": 2,
    "totalCount": 15,
    "alarms": [
      {
        "alarmId": 301,
        "type": "FEEDBACK",
        "title": "새로운 피드백이 도착했습니다.",
        "body": "거실 - '너무 더워요'",
        "isRead": false,
        "placeId": 501,
        "createdAt": "2026-05-17T21:05:04Z"
      },
      {
        "alarmId": 300,
        "type": "SENSOR_ALERT",
        "title": "불쾌지수 경고",
        "body": "거실의 불쾌지수가 80을 초과했습니다.",
        "isRead": true,
        "placeId": 501,
        "createdAt": "2026-05-17T20:30:00Z"
      }
    ]
  }
}
```

**실패 케이스 - 401 Unauthorized**
```json
{ "success": false, "message": "인증 정보가 유효하지 않습니다. 다시 로그인해주세요.", "data": null }
```

**실패 케이스 - 500 Internal Server Error**
```json
{ "success": false, "message": "서버 내부 오류가 발생했습니다.", "data": null }
```

## 📍 21 알람 삭제 API

- **설명:** 특정 알람을 삭제하는 API입니다. 단건 삭제 및 전체 삭제를 지원합니다. 본인의 알람만 삭제 가능합니다. (인증 필요)
- **헤더:** `Authorization: Bearer {accessToken}`

**Parameter**

| 필드 이름 | 필수 여부 | 타입 | 설명 |
| --- | --- | --- | --- |
| alarmId | 선택 | Long | 삭제할 알람 ID (미입력 시 전체 삭제) |

**Request Body**

| 필드 이름 | 필수 여부 | 타입 | 설명 |
| --- | --- | --- | --- |

**Request Sample**

```
DELETE /api/alarm?alarmId=301   ← 단건 삭제
DELETE /api/alarm                ← 전체 삭제
```

**응답코드**

| 성공 여부 | 응답 코드 | 설명 |
| --- | --- | --- |
| 성공 | 200 | 알람 삭제 완료 |
| 실패 | 401 | 인증 실패 |
| 실패 | 403 | 본인 알람이 아님 |
| 실패 | 404 | 존재하지 않는 알람 |
| 실패 | 500 | 서버 내부 오류 |

**Response Body**

| 필드 이름 | 필수 여부 | 타입 | 설명 |
| --- | --- | --- | --- |
| deletedCount | 필수 | Int | 삭제된 알람 수 |

**JSON 형식 응답**

```json
{
  "success": true,
  "message": "알람이 삭제되었습니다.",
  "data": {
    "deletedCount": 1
  }
}
```

**실패 케이스 - 401 Unauthorized**
```json
{ "success": false, "message": "인증 정보가 유효하지 않습니다. 다시 로그인해주세요.", "data": null }
```

**실패 케이스 - 403 Forbidden**
```json
{ "success": false, "message": "권한이 없습니다.", "data": null }
```

**실패 케이스 - 404 Not Found**
```json
{ "success": false, "message": "존재하지 않는 알람입니다.", "data": null }
```

**실패 케이스 - 500 Internal Server Error**
```json
{ "success": false, "message": "서버 내부 오류가 발생했습니다.", "data": null }
```
