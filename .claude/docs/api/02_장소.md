# 📍 장소 API 명세

## 📍 07 새로운 장소 등록 API

- **설명:** 사용자가 IoT 기기를 배치하고 관리할 새로운 장소(예: 집, 매장, 회사 등)를 시스템에 등록하는 API입니다. 등록 시 장소의 이름과 함께 공간의 유형(HOME, STORE, COMPANY)을 지정하여 맞춤형 서비스 환경을 제공합니다. (인증된 회원만 접근 가능)
- **헤더:** `Authorization: Bearer {accessToken}`

**Parameter**

| 필드 이름 | 필수 여부 | 타입 | 설명 |
| --- | --- | --- | --- |

**Request Body**

| 필드 이름 | 필수 여부 | 타입 | 설명 |
| --- | --- | --- | --- |
| name | 필수 | String | 사용자가 설정할 장소의 이름 (예: 우리집, 상가 1호점 등) |
| type | 필수 | String | 공간 유형 (HOME: 집, STORE: 매장, COMPANY: 회사) |

**Request Sample**

```json
{
  "name": "우리집",
  "type": "HOME"
}
```

**응답코드**

| 성공 여부 | 응답 코드 | 설명 |
| --- | --- | --- |
| 성공 | 200 | 새로운 장소 등록 성공 및 고유 식별자 발급 완료 |
| 실패 | 400 | 잘못된 요청 (장소 이름 누락 또는 정의되지 않은 공간 유형) |
| 실패 | 401 | 인증 실패 (유효하지 않거나 만료된 AccessToken) |
| 실패 | 404 | 존재하지 않는 회원 (토큰에서 추출한 userId가 DB에 없음) |
| 실패 | 500 | 서버 내부 오류 |

**Response Body**

| 필드 이름 | 필수 여부 | 타입 | 설명 |
| --- | --- | --- | --- |
| placeId | 필수 | Int | 새로 생성된 장소의 고유 식별자 ID |
| name | 필수 | String | 등록된 장소의 이름 |
| type | 필수 | String | 등록된 공간 유형 (HOME, STORE, COMPANY) |
| createdAt | 필수 | DateTime | 장소 생성 완료 시간 (ISO 8601) |

**JSON 형식 응답**

```json
{
  "success": true,
  "message": "API 호출 성공",
  "data": {
    "placeId": 501,
    "name": "우리집",
    "type": "HOME",
    "createdAt": "2026-05-17T21:05:04Z"
  }
}
```

**실패 케이스 - 400 Bad Request**
```json
{ "success": false, "message": "요청 형식이 올바르지 않습니다.", "data": null }
```

**실패 케이스 - 401 Unauthorized**
```json
{ "success": false, "message": "인증 정보가 유효하지 않습니다. 다시 로그인해주세요.", "data": null }
```

**실패 케이스 - 404 Not Found**
```json
{ "success": false, "message": "존재하지 않는 회원 정보입니다.", "data": null }
```

**실패 케이스 - 500 Internal Server Error**
```json
{ "success": false, "message": "서버 내부 오류가 발생했습니다.", "data": null }
```

## 📍 08 페어링 코드 생성 API

- **설명:** 공기계 앱과 장소를 연결하기 위한 일회용 페어링 코드를 생성하는 API입니다. 생성된 코드는 Redis에 일정 시간(10분) 동안 저장되며, 공기계 앱에서 해당 코드를 입력하면 장소와 기기가 연결됩니다. (ADMIN 권한 필요)
- **헤더:** `Authorization: Bearer {accessToken}`

**Parameter**

| 필드 이름 | 필수 여부 | 타입 | 설명 |
| --- | --- | --- | --- |
| placeId | 필수 | Int | 페어링할 장소의 고유 ID |

**Request Body**

| 필드 이름 | 필수 여부 | 타입 | 설명 |
| --- | --- | --- | --- |

**Request Sample**

```
POST /api/place/pairing/code?placeId=501
```

**응답코드**

| 성공 여부 | 응답 코드 | 설명 |
| --- | --- | --- |
| 성공 | 200 | 페어링 코드 생성 완료 |
| 실패 | 401 | 인증 실패 |
| 실패 | 403 | ADMIN 권한 없음 |
| 실패 | 404 | 존재하지 않는 장소 또는 회원 |
| 실패 | 500 | 서버 내부 오류 |

**Response Body**

| 필드 이름 | 필수 여부 | 타입 | 설명 |
| --- | --- | --- | --- |
| pairingCode | 필수 | String | 생성된 6자리 페어링 코드 |
| expiresAt | 필수 | DateTime | 코드 만료 시간 (생성 후 10분, ISO 8601) |

**JSON 형식 응답**

```json
{
  "success": true,
  "message": "API 호출 성공",
  "data": {
    "pairingCode": "A3F7K2",
    "expiresAt": "2026-05-17T21:15:00Z"
  }
}
```

**실패 케이스 - 401 Unauthorized**
```json
{ "success": false, "message": "인증 정보가 유효하지 않습니다. 다시 로그인해주세요.", "data": null }
```

**실패 케이스 - 403 Forbidden**
```json
{ "success": false, "message": "권한이 없습니다.", "data": null }
```

**실패 케이스 - 404 Not Found**
```json
{ "success": false, "message": "존재하지 않는 장소 정보입니다.", "data": null }
```

**실패 케이스 - 500 Internal Server Error**
```json
{ "success": false, "message": "서버 내부 오류가 발생했습니다.", "data": null }
```

## 📍 09 페어링 코드 입력 API

- **설명:** 공기계 앱에서 관리자가 생성한 페어링 코드를 입력하여 해당 장소와 기기를 연결하는 API입니다. 코드 검증 후 device 테이블에 KIOSK 타입으로 기기가 등록되며, WebSocket 연결을 위한 appToken이 발급됩니다.
- **헤더:** `Authorization: Bearer {accessToken}`

**Parameter**

| 필드 이름 | 필수 여부 | 타입 | 설명 |
| --- | --- | --- | --- |

**Request Body**

| 필드 이름 | 필수 여부 | 타입 | 설명 |
| --- | --- | --- | --- |
| pairingCode | 필수 | String | 관리자로부터 전달받은 6자리 페어링 코드 |
| deviceName | 필수 | String | 공기계 기기 이름 (예: 거실 공기계) |

**Request Sample**

```json
{
  "pairingCode": "A3F7K2",
  "deviceName": "거실 공기계"
}
```

**응답코드**

| 성공 여부 | 응답 코드 | 설명 |
| --- | --- | --- |
| 성공 | 200 | 페어링 완료 및 기기 등록 성공 |
| 실패 | 400 | 코드 누락 또는 만료된 코드 |
| 실패 | 401 | 인증 실패 |
| 실패 | 404 | 존재하지 않는 회원 |
| 실패 | 500 | 서버 내부 오류 |

**Response Body**

| 필드 이름 | 필수 여부 | 타입 | 설명 |
| --- | --- | --- | --- |
| deviceId | 필수 | String | 등록된 기기의 고유 ID |
| placeId | 필수 | Int | 연결된 장소 ID |
| appToken | 필수 | String | WebSocket 연결에 사용할 앱 토큰 |

**JSON 형식 응답**

```json
{
  "success": true,
  "message": "페어링이 완료되었습니다.",
  "data": {
    "deviceId": "kiosk_a3f7k2_1716",
    "placeId": 501,
    "appToken": "eyJhbGciOiJIUzI1NiJ9..."
  }
}
```

**실패 케이스 - 400 Bad Request**
```json
{ "success": false, "message": "페어링 코드가 만료되었거나 유효하지 않습니다.", "data": null }
```

**실패 케이스 - 401 Unauthorized**
```json
{ "success": false, "message": "인증 정보가 유효하지 않습니다. 다시 로그인해주세요.", "data": null }
```

**실패 케이스 - 500 Internal Server Error**
```json
{ "success": false, "message": "서버 내부 오류가 발생했습니다.", "data": null }
```

## 📍 10 관리자 코드 생성 API

- **설명:** 특정 장소에 새로운 관리자를 초대하기 위한 일회용 관리자 초대 코드를 생성하는 API입니다. 생성된 코드는 Redis에 저장되며(30분 유효), 초대받은 사용자가 코드를 입력하면 해당 장소의 ADMIN 권한이 부여됩니다. (기존 ADMIN 권한 필요)
- **헤더:** `Authorization: Bearer {accessToken}`

**Parameter**

| 필드 이름 | 필수 여부 | 타입 | 설명 |
| --- | --- | --- | --- |
| placeId | 필수 | Int | 관리자를 초대할 장소의 고유 ID |

**Request Body**

| 필드 이름 | 필수 여부 | 타입 | 설명 |
| --- | --- | --- | --- |

**Request Sample**

```
POST /api/place/admin/code?placeId=501
```

**응답코드**

| 성공 여부 | 응답 코드 | 설명 |
| --- | --- | --- |
| 성공 | 200 | 관리자 초대 코드 생성 완료 |
| 실패 | 401 | 인증 실패 |
| 실패 | 403 | ADMIN 권한 없음 |
| 실패 | 404 | 존재하지 않는 장소 또는 회원 |
| 실패 | 500 | 서버 내부 오류 |

**Response Body**

| 필드 이름 | 필수 여부 | 타입 | 설명 |
| --- | --- | --- | --- |
| adminCode | 필수 | String | 생성된 8자리 관리자 초대 코드 |
| expiresAt | 필수 | DateTime | 코드 만료 시간 (생성 후 30분, ISO 8601) |

**JSON 형식 응답**

```json
{
  "success": true,
  "message": "API 호출 성공",
  "data": {
    "adminCode": "RB7X2K9P",
    "expiresAt": "2026-05-17T21:35:00Z"
  }
}
```

```json
{ "success": false, "message": "인증 정보가 유효하지 않습니다. 다시 로그인해주세요.", "data": null }
```

**실패 케이스 - 403 Forbidden**
```json
{ "success": false, "message": "권한이 없습니다.", "data": null }
```

**실패 케이스 - 404 Not Found**
```json
{ "success": false, "message": "존재하지 않는 장소 정보입니다.", "data": null }
```

**실패 케이스 - 500 Internal Server Error**

```json
{ "success": false, "message": "서버 내부 오류가 발생했습니다.", "data": null }
```

## 📍 11 관리자 코드 입력 API

- **설명:** 초대받은 사용자가 관리자 초대 코드를 입력하여 해당 장소의 ADMIN 권한을 획득하는 API입니다. 코드 검증 후 user_place_mapping 테이블에 ADMIN 레코드가 생성됩니다.
- **헤더:** `Authorization: Bearer {accessToken}`

**Parameter**

| 필드 이름 | 필수 여부 | 타입 | 설명 |
| --- | --- | --- | --- |

**Request Body**

| 필드 이름 | 필수 여부 | 타입 | 설명 |
| --- | --- | --- | --- |
| adminCode | 필수 | String | 전달받은 8자리 관리자 초대 코드 |

**Request Sample**

```json
{
  "adminCode": "RB7X2K9P"
}
```

**응답코드**

| 성공 여부 | 응답 코드 | 설명 |
| --- | --- | --- |
| 성공 | 200 | ADMIN 권한 부여 완료 |
| 실패 | 400 | 코드 누락 또는 만료된 코드 |
| 실패 | 401 | 인증 실패 |
| 실패 | 409 | 이미 해당 장소의 관리자로 등록된 경우 |
| 실패 | 500 | 서버 내부 오류 |

**Response Body**

| 필드 이름 | 필수 여부 | 타입 | 설명 |
| --- | --- | --- | --- |
| placeId | 필수 | Int | 권한이 부여된 장소 ID |
| placeName | 필수 | String | 장소 이름 |
| accessLevel | 필수 | String | 부여된 권한 (ADMIN) |

**JSON 형식 응답**

```json
{
  "success": true,
  "message": "관리자 권한이 부여되었습니다.",
  "data": {
    "placeId": 501,
    "placeName": "우리집",
    "accessLevel": "ADMIN"
  }
}
```

**실패 케이스 - 400 Bad Request**
```json
{ "success": false, "message": "관리자 코드가 만료되었거나 유효하지 않습니다.", "data": null }
```

**실패 케이스 - 401 Unauthorized**
```json
{ "success": false, "message": "인증 정보가 유효하지 않습니다. 다시 로그인해주세요.", "data": null }
```

**실패 케이스 - 409 Conflict**
```json
{ "success": false, "message": "이미 해당 장소의 관리자로 등록되어 있습니다.", "data": null }
```

**실패 케이스 - 500 Internal Server Error**
```json
{ "success": false, "message": "서버 내부 오류가 발생했습니다.", "data": null }
```
